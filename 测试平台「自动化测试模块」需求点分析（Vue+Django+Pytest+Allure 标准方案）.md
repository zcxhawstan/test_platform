# 测试平台「自动化测试模块」需求点分析（Vue+Django+Pytest+Allure 标准方案）

我按照**企业级测试平台标准设计规范**，结合你现有 `Vue+Django+Pytest` 技术栈，针对**Python 脚本自动化执行、环境配置、日志查看、Allure 报告**核心需求，做完整、可落地、可直接开发的需求拆解。

整体定位：**Web 可视化 Python 自动化任务管理平台**，支持任务创建→多环境运行→实时日志→报告归档→历史查询全流程。

---

## 一、模块整体目标

1. 无需登录服务器，在前端页面一键创建、执行 Python 自动化脚本任务

2. 支持**多环境配置**（测试/预发/生产/自定义环境）动态切换运行

3. 实时查看任务执行日志，支持日志下载

4. 自动生成、展示、归档、下载 Allure 测试报告

5. 任务可管理、可重跑、可查看历史记录

6. 与现有 Django 后端、Vue 前端、Pytest 框架无缝集成

---

## 二、核心功能模块拆解（共 5 大模块）

### 模块 1：自动化任务列表（首页）

**页面功能**：所有自动化任务的总览入口

- 任务列表展示：任务ID、任务名称、创建人、创建时间、最近执行状态、最近执行时间、操作按钮

- 筛选功能：按任务名称、执行状态、环境、创建时间筛选

- 搜索功能：支持任务名称模糊搜索

- 分页功能：支持分页展示大量任务

- 操作按钮：**创建任务、执行任务、查看日志、查看报告、编辑、删除、重跑**

**状态定义**：

- 待执行

- 执行中

- 执行成功

- 执行失败

- 执行异常

---

### 模块 2：自动化任务创建/编辑（核心页面）

**页面功能**：可视化配置 Python 自动化执行任务，支持多环境

#### 2.1 基础信息配置

- 任务名称（必填）

- 任务描述（选填）

- 归属项目/模块（可选，用于分类）

- 创建人/更新人自动记录

#### 2.2 脚本配置（核心）

- 脚本来源：

    1. 平台内置脚本（平台统一管理的 pytest 用例）

    2. 自定义上传 Python 脚本（.py 文件）

    3. Git 仓库脚本地址（进阶，支持拉取最新代码执行）

- 脚本执行命令：默认 `pytest xxx.py --alluredir=./result`，支持自定义命令

- 执行目录：自动分配，无需用户配置

- 用例筛选：支持按模块、标签、用例名称筛选执行（Pytest 原生能力）

#### 2.3 环境配置（必须支持多环境）

- 环境选择：下拉框（测试环境/预发环境/生产环境/自定义环境）

- 环境变量配置：

    - 域名/IP：`BASE_URL`

    - 数据库配置

    - 账号密码

    - 其他自定义键值对

- 环境配置支持：**新增、编辑、删除、保存为模板**

- 运行时动态注入环境变量到 Python 脚本中

#### 2.4 执行配置

- 执行方式：手动执行 / 定时执行（可选，进阶需求）

- 失败重跑：开启/关闭，重跑次数

- 报告生成：默认开启 Allure 报告

- 执行超时时间：防止死循环（如 30min / 60min）

#### 2.5 保存与校验

- 表单必填项校验

- 保存为草稿 / 直接创建

- 编辑时回显所有配置

---

### 模块 3：任务执行与实时日志查看

**页面功能**：执行任务并实时输出控制台日志

#### 3.1 任务执行

- 点击「执行」→ 后端异步启动任务（Django + Celery 必备）

- 前端显示执行中状态 + 进度条

#### 3.2 实时日志（核心体验）

- 日志实时滚动输出（Websocket 最佳）

- 日志样式：标准命令行样式，区分 INFO/ERROR/PASS/FAIL

- 日志搜索：关键词搜索

- 日志清空/刷新/暂停滚动

- 日志下载：执行完成后下载完整日志文件（.log）

- 日志持久化：后端存储，可随时回看

#### 3.3 执行中断

- 支持手动停止正在执行的任务

- 停止后日志终止，标记为“已中断”

---

### 模块 4：Allure 测试报告展示与管理

**页面功能**：自动生成、在线查看、归档 Allure 报告（行业标准）

#### 4.1 报告生成规则

- 执行完成后**自动生成** Allure 报告

- 报告与任务执行记录一对一绑定

- 报告永久存储（支持清理策略）

#### 4.2 报告查看方式

1. **在线预览**：Vue 页面内嵌 Allure 报告（无需打开新页面）

2. **新窗口打开**：完整 Allure 交互报告

3. **报告下载**：下载 HTML 压缩包，本地打开

#### 4.3 报告展示数据（标准 Allure 字段）

- 用例总数、通过数、失败数、跳过数、通过率

- 执行耗时

- 用例明细：步骤、断言、截图、错误信息

- 趋势图（历史执行对比）

#### 4.4 报告管理

- 按任务、按时间筛选历史报告

- 报告重生成

- 报告删除/清理

---

### 模块 5：任务执行历史记录

**页面功能**：查看某一任务的所有历史执行记录

- 执行记录列表：执行时间、执行人、环境、状态、耗时、操作

- 每条记录可：查看日志、查看报告、重跑本次

- 支持按状态、时间筛选

- 支持批量删除历史记录

## 三、非功能需求（平台级标准）

### 1. 异步执行（必须）

- 任务不能同步执行，必须使用 **Django + Celery** 异步执行

- 防止前端页面阻塞、超时

### 2. 环境隔离

- 不同环境执行互不影响

- 环境变量不硬编码，统一平台管理

- 敏感信息（密码）加密存储

### 3. 文件管理

- 脚本文件、日志文件、Allure 报告统一存储

- 支持本地存储 / MinIO / 阿里云 OSS

### 4. 权限控制

- 谁可以创建任务

- 谁可以执行任务

- 谁可以删除任务

- 谁可以查看生产环境报告

### 5. 稳定性

- 任务异常自动捕获

- 超时自动终止

- 服务重启后任务可恢复

### 6. 兼容性

- 支持 Python 3.8+

- 支持所有 Pytest 插件

- 支持 Allure 2.20+
